# null-logs アカウント 情報発信戦略書

**アカウント名:** null-logs
**プラットフォーム:** X（旧Twitter）
**ステータス:** 運用中

---

## 1. アカウント定義

### 1.1 ポジショニング

> 「K.の人間としての全部がある場所」

K.が経営者・プロフェッショナルとしての「絞られた人格」を持つのに対し、null-logsはその人間としての興味・知的好奇心・感情反応が全部そのまま出る**アンフィルタードなアウトプット空間**。

KPIや目標を持たない。好きなことを好きなタイミングで投稿する。それがこのアカウントの価値。

### 1.2 役割の2軸

| 役割 | 説明 |
|---|---|
| **アウトプット置き場** | 興味のあることを思ったまま投稿する。完成度不要 |
| **K.の援護射撃** | K.の投稿にいいね・リポスト・引用でエンゲージメントを底上げする |

### 1.3 コンテンツジャンル（制限なし）

以下はあくまで自然発生するジャンル一覧。義務ではない。

| ジャンル | 内容例 |
|---|---|
| **AI・テック** | LLMの動向、Claude Code実装メモ、エージェント設計 |
| **投資・ファイナンス** | 金利、資産形成、リスク管理、相場感 ※K.から移行 |
| **経営・ビジネス** | 読んだ記事・書籍の断片的な感想や気づき |
| **宇宙** | 宇宙開発ニュース、物理学、スケールの大きい話 |
| **歴史・哲学** | 歴史的事件の考察、哲学的な問いへの反応 |
| **NBA** | 試合感想、選手分析、シーズン動向 |
| **海外サッカー** | 試合感想、戦術、移籍情報への反応 |
| **日常・雑感** | 何でも |

## 2. 運用方針

### 2.1 基本スタンス

- **投稿ハードルをゼロにする** — 140字以下の雑メモでOK。完成度を求めない
- **数字を追わない** — インプレッション・フォロワー増加はKPIとして管理しない
- **K.との連動** — K.が投稿したら基本的にいいね。反響が大きそうなものはリポストも行う

### 2.2 K.援護射撃プロトコル

| アクション | タイミング |
|---|---|
| **いいね** | K.の全投稿に原則実施 |
| **リポスト** | 事業進捗ログ・AIツール実況など拡散価値が高い投稿 |
| **引用リポスト** | null-logsとして補足コメントを添えられる場合 |

> 注意: K.との名寄せが明確になる言及・リプライは避ける。あくまで一般フォロワーとして自然に動く。

### 2.3 投稿頻度

- 義務なし。思ったときに投稿する
- 最低ラインとして**週3〜5本**あればアカウントが死なない程度には維持される

### 2.4 文体・トーン定義

過去の実際のポストから抽出した文体ルール。コンテンツ生成時はこのトーンを維持する。

#### 基本原則

| 原則 | 内容 |
|---|---|
| **短く切る** | 1〜8文。余計な説明を足さない |
| **断言しない** | 「〜かなと」「〜気がする」「〜待とうかな」で終わる。確信でなく思考メモとして出す |
| **学習中をそのまま出す** | 理解しきれていなくても投稿する。「今のところの理解。」「〜で学習中。」がそのまま価値になる |
| **ハッシュタグなし** | 一切使わない |
| **口語** | 「結局」「でも」「一旦」「かなり」など自然な話し言葉 |

#### 投稿パターン

**パターンA｜ニュース引用 + 自分のコメント**
```
>[引用元テキスト]

[自分の解釈・見解。1〜3文]
```

**パターンB｜読書・学習ログ**
```
[本のタイトルや学習テーマ]
[一言で言うと何だったか]

[自分への翻訳・感想・疑問]
```

**パターンC｜比喩・アナロジーで整理**
```
[テーマ]を[身近なもの]に例えるとわかりやすい。

今のところの理解。

[A]: [説明]（[具体名]）
[B]: [説明]（[具体名]）
...
```

**パターンD｜意思決定メモ**
```
[やりたいこと・ポジション検討]

[でも/ただ、懸念点や待ち理由]
```

#### 参考ポスト（実例）

```
サイコロジー・オブ・マネー読了。
2026年最初の書籍。

結局モーガン・ハウセルもインデックスしかやってないとのこと。

でも個別株をやることによって各業界や経済に興味が出て、世界を理解することにつながるので、
40-50代までにある程度理解し、徐々にインデックスに移行して行くのが良いのかなと
```

```
半導体業界についてGeminiで学習中。
「本づくり」に例えるとわかりやすい。

今のところの理解。

紙: 信越化学, SUMCO（材料）
印刷機: ASML（露光装置）
印刷所: TSM（製造）
本を書く人: NVIDIA, AMD（設計）
本屋: OpenAI, Google, Amazon（AIサービス）
```

```

```

## 3. 投資・ファイナンスコンテンツの移行

K.から移行した投資・ファイナンス系コンテンツはnull-logsが主戦場となる。

**移行理由:**
- K.のブランド（経営者1年目のリアルログ）とジャンルが微妙にズレる
- null-logsのほうが「好きなことを語る場所」として自然にフィットする
- K.のエンゲージメントリソースを本来の柱（事業進捗・AI実装）に集中させる

**投稿スタイル（投資・ファイナンス）:**
- 実践的なアドバイスよりも「自分がどう考えて動いたか」の記録スタイル
- 断言形式よりも思考メモ形式のほうがnull-logsのトーンに合う
