コンテンツを生成します。

以下の情報を確認してから開始してください：

**引数**: `$ARGUMENTS`
（フォーマット: `[事業名] [コンテンツ種別]` 例: `dober x-post` / `kyohei instagram` / `dober blog`）

---

## 実行手順

1. **コンテキスト確認**
   - 事業/クライアントの CONTEXT.md または BRIEF.md を読む
   - トンマナ・禁止表現を把握する

2. **プラットフォーム特定**
   - x-post → `agents/specialists/marketing/x-specialist.md` のフォーマット
   - instagram → `agents/specialists/marketing/instagram-specialist.md` のフォーマット
   - threads → `agents/specialists/marketing/threads-specialist.md` のフォーマット
   - blog → `agents/specialists/contents/blog-writer.md` の手順
   - lp / copy → `agents/specialists/contents/copywriter.md` の手順

3. **コンテンツ作成**
   - `agents/specialists/contents/sns-writer.md` または該当Writerの役割で作成
   - 複数案（最低3案）を提示する

4. **編集・校正**
   - `agents/specialists/contents/editor.md` のチェックリストで確認

5. **保存**
   - 下書き: `businesses/[事業名]/content/drafts/YYYY-MM-DD_[種別].md`
   - 公開後: `businesses/[事業名]/content/published/` に移動
