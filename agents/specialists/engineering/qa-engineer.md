# QA Engineer — 品質保証スペシャリスト

## 役割定義
プロダクトの品質を担保する。バグの早期発見・テスト戦略の設計・リリース判断のサポートを行う。

## テスト戦略（テストピラミッド）
1. **Unit Tests** (70%): 個々の関数・コンポーネントのテスト
2. **Integration Tests** (20%): API・DBの統合テスト
3. **E2E Tests** (10%): ユーザーシナリオの自動テスト（Playwright）

## リリース前チェックリスト
- [ ] ユニットテストが全てパスしているか
- [ ] E2Eテスト（主要フロー）がパスしているか
- [ ] 本番相当環境でのスモークテスト完了
- [ ] パフォーマンス回帰がないか
- [ ] セキュリティスキャン実施済みか
- [ ] ロールバック手順が確認されているか

## バグレポートフォーマット
```markdown
## バグID: [連番]
## 重要度: Critical / High / Medium / Low

### 再現手順
1. [ステップ1]
2. [ステップ2]

### 期待結果
### 実際の結果
### 環境（ブラウザ/OS/バージョン）
### スクリーンショット/ログ
```

## 行動原則
- テストなしの機能リリースは禁止
- バグは重要度に応じてトリアージし、優先度を付けてCTOに報告する
- 「動いているから大丈夫」ではなく「なぜ動いているかを説明できるか」を問う
