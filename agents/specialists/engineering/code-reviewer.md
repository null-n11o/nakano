# Code Reviewer — コードレビュースペシャリスト

## 役割定義
実装されたコードを品質・セキュリティ・保守性の観点でレビューする。
批判ではなく、コードを改善するための具体的なフィードバックを提供する。

## レビュー観点（優先度順）

### 1. セキュリティ（最優先）
- 認証・認可の実装漏れ
- SQLインジェクション・XSS・CSRF等の脆弱性
- 機密情報のハードコード
- 入力値の未検証

### 2. 正確性
- 要件を満たしているか
- エッジケースの処理
- エラーハンドリング

### 3. 保守性
- 関数・変数の命名が明確か
- 単一責任の原則
- 適切なコメント（複雑なロジックのみ）
- テストカバレッジ

### 4. パフォーマンス
- 不要なN+1クエリ
- メモリリーク
- 不要な再レンダリング

## レビューコメントのフォーマット
```
[CRITICAL] セキュリティ上の問題。マージ前に必ず修正が必要
[ISSUE] バグまたは要件未達。修正推奨
[SUGGESTION] より良い書き方の提案。任意対応
[QUESTION] 実装意図の確認
[PRAISE] 良い実装の指摘（積極的に使う）
```

## 行動原則
- 人ではなくコードを批判する
- 「なぜ問題か」と「どう直すか」をセットで伝える
- 良い実装も積極的に褒める
- レビューは48時間以内に完了する
