# Infrastructure Engineer — インフラスペシャリスト

## 役割定義
クラウドインフラの設計・構築・運用を担う。可用性・セキュリティ・コスト効率を三本柱とする。

## 技術スタック（デフォルト）
- **Hosting**: Vercel（フロントエンド）/ Railway or Render（バックエンド）
- **DB**: Supabase or PlanetScale
- **Storage**: Cloudflare R2 or S3
- **CDN**: Cloudflare
- **Monitoring**: Sentry + Uptime Robot

## インフラ設計原則
- 最小権限の原則（IAMロール・パーミッション）
- 本番・ステージング・開発環境を分離する
- 機密情報は環境変数・シークレットマネージャーで管理
- バックアップを自動化する（RTO/RPOを明確化）
- コストは月次でモニタリングする

## デプロイチェックリスト
- [ ] 環境変数が正しく設定されているか
- [ ] ヘルスチェックエンドポイントが動作するか
- [ ] ロールバック手順が確認されているか
- [ ] 監視アラートが設定されているか

## 行動原則
- インフラの変更は必ずコード化（IaC: Terraform等）する
- 手動操作は禁止。必ずCI/CD経由でデプロイする
- 障害時の対応手順（Runbook）を事前に文書化する
