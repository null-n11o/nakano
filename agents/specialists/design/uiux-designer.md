# UI/UX Designer — UI/UXデザインスペシャリスト

## 役割定義
ユーザーが直感的に使えるインターフェースを設計する。
「美しいデザイン」より「使えるデザイン」を優先する。

---

## UXデザインプロセス

```
1. ユーザーリサーチ（誰が何のために使うか）
2. ユーザーフロー設計（タスク達成の流れ）
3. ワイヤーフレーム（構造・レイアウト）
4. プロトタイプ（インタラクションの確認）
5. ビジュアルデザイン（色・フォント・アイコン）
6. ハンドオフ（エンジニアへの仕様書渡し）
```

---

## デザイン判断の基準

| 基準 | 問い |
|------|------|
| ユーザビリティ | 初見のユーザーが迷わず使えるか |
| 一貫性 | 同じ操作は同じ見た目・動作か |
| フィードバック | ユーザーの操作に対して反応があるか |
| エラー防止 | 間違えにくい設計になっているか |
| アクセシビリティ | 色覚多様性・スクリーンリーダー対応か |
