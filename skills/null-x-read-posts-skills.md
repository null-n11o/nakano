# null Reading Log Skill

ケンタロウのnullアカウント用の読書・視聴ログ投稿を作成するスキル。

## コンセプト

nullは「ポストシンギュラリティ時代に生き残るための思考・読書ログ」をテーマにしたアカウント。
AI、宇宙、物理、生物、経済、歴史、哲学、経営、投資などの分野を横断して、インプットしたものをアウトプットする外部記憶装置兼信頼蓄積装置として機能する。

## 投稿フォーマット

### 1. トップ投稿

```
**『タイトル』**

[1-2文の第一印象・総評を思考ログ風に]

[なぜこの本/映画が価値があるかを2-3文で]

[どんな人が読むべきか、または自分にとっての位置づけを1文で]
```

**トーンのルール:**
- 「〜だ。」「〜である。」は使わない
- 「〜だった。」「〜になる。」「〜と思う。」など、自分の思考ログとして書く
- 断定的な評論家口調ではなく、個人の気づき・感想として書く

**良い例:**
- 「投資本だと思って読み始めたが、完全に人生哲学の本だった。」
- 「『お金とどう向き合うか』は『時間とどう向き合うか』と同義で、つまり『人生をどうコントロールするか』の話になる。」
- 「投資テクニックを学びたいなら他の本を読んだ方がいい。『なぜ自分は投資をするのか』という問いに向き合いたいなら読むべき一冊。」

**悪い例:**
- 「これは投資本ではない。人生哲学の本だ。」（断定的すぎる）
- 「本書は〜について述べている。」（評論家口調）

### 2. スレッド（ぶら下げ）

ユーザーから提供されたメモ・箇条書きを、**全て**使ってスレッド形式に変換する。

**構成ルール:**
- 提供されたメモの内容は省略せず、全て使い切る
- 論理的にグルーピングできる場合は【見出し】で区切る（例: 【富】【幸福】【明晰に考える方法】）
- グルーピングが不要な場合は番号のみで進める（①②③...）
- 各ツイートは1つの学び・気づきに絞る
- 原文の意味を保ちつつ、読みやすく整形する

**トーンのルール:**
- メモの内容をそのまま活かす（過度に言い換えない）
- 引用や格言はそのまま使ってOK
- 自分の解釈を加える場合は「〜ということ。」「〜だと思った。」など思考ログ風に

**フォーマット例:**

```
**①**
[学び1の内容]

**②**
[学び2の内容]

**【セクション名】**

**③**
[学び3の内容]
```

## 入力形式

ユーザーは以下の形式でメモを提供する:
- 箇条書きのメモ
- コピペした引用集
- 自分の感想メモ
- 上記の混合

## 出力形式

```
## トップ投稿

**『[タイトル]』**

[トップ投稿の本文]

---

## スレッド（ぶら下げ）

**①**
[内容]

**②**
[内容]

...
```

## 注意事項

1. **メモは全て使う**: ユーザーが提供した内容は省略せず、全てスレッドに含める
2. **トーンの一貫性**: 評論家ではなく、個人の思考ログとして書く
3. **過度な装飾を避ける**: シンプルに、読みやすく
4. **絵文字は使わない**: nullのトーンに合わない
4. **ハッシュタグは使わない**: nullのトーンに合わない
