# スキルの配布・共有ガイド

---

## 配布方法（2026年1月時点）

### 個人ユーザーがスキルを入手する方法

1. スキルフォルダをダウンロード
2. フォルダを ZIP 圧縮
3. Claude.ai の設定 > Capabilities > Skills からアップロード
4. または Claude Code のスキルディレクトリに配置

### 組織レベルのスキル

- 管理者がワークスペース全体にスキルをデプロイ可能
- 自動アップデート対応
- 一元管理

---

## API経由でのスキル利用

プログラムによるスキル管理・実行：

```
POST /v1/skills          # スキルのリスト・管理
Messages API             # container.skills パラメータでスキルを追加
Claude Agent SDK         # カスタムエージェント構築に使用
```

| ユースケース | 推奨サーフェス |
|------------|---------------|
| ユーザーが直接スキルを使う | Claude.ai / Claude Code |
| 開発中の手動テスト | Claude.ai / Claude Code |
| 個別・アドホックなワークフロー | Claude.ai / Claude Code |
| スキルをプログラムで使うアプリ | API |
| 大規模な本番デプロイ | API |
| 自動パイプライン・エージェントシステム | API |

## GitHub での推奨配布方法

### 1. GitHub でホスト

```
your-skill-repo/
├── README.md          # 人間向け説明（スキルフォルダ内には入れない）
├── your-skill-name/
│   ├── SKILL.md
│   ├── references/
│   └── scripts/
└── examples/          # スクリーンショット・使用例
```

**README に含めるべき内容:**
- インストール手順
- 使用例（スクリーンショット付き）
- MCP との組み合わせ方（MCPスキルの場合）

### 2. MCP ドキュメントと連携させる

MCP スキルの場合、MCP のドキュメントにスキルへのリンクを追加：

```markdown
## スキルで自動化する

[スキル名] スキルをインストールすると、[サービス名] の主要ワークフローが
自動化されます。

[スキルをインストール](https://github.com/yourcompany/skills)
```

### 3. インストールガイドを作成

```markdown
## [サービス名] スキルのインストール

1. スキルをダウンロード:
   `git clone https://github.com/yourcompany/skills`

2. Claude にインストール:
   Claude.ai > 設定 > スキル > 「スキルをアップロード」

3. スキルを有効化:
   [サービス名] スキルのトグルをオン

4. テスト:
   「[サービス名] で新しいプロジェクトを設定して」と聞いてみる
```

## スキルの説明文（価値の伝え方）

### 機能ではなく成果にフォーカスする

**良い例:**
> 「ProjectHub スキルを使えば、ページ・データベース・テンプレートを含む
> 完全なプロジェクトワークスペースを数秒でセットアップできます。
> 従来の30分の手動作業が不要になります。」

**悪い例:**
> 「ProjectHub スキルは YAML frontmatter と Markdown 指示を含むフォルダで、
> MCP サーバーツールを呼び出します。」

### MCP + スキルのストーリーを伝える

> 「MCP サーバーが Claude に Linear プロジェクトへのアクセスを提供します。
> スキルがチームのスプリント計画ワークフローを Claude に教えます。
> 組み合わせることで、AI駆動のプロジェクト管理が実現します。」

## オープンスタンダード

Anthropic はスキルをオープンスタンダードとして公開している。MCPと同様に、スキルはツールやプラットフォームを超えて移植可能であるべきという考え方。同じスキルが Claude でも他のAIプラットフォームでも動作することを目指している。
