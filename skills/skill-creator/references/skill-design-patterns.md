# スキル設計パターン

PDF「The Complete Guide to Building Skills for Claude」に基づくパターン集。

---

## 3つのユースケースカテゴリ

### カテゴリ1: ドキュメント・アセット生成

**目的:** 一貫した高品質のアウトプットを生成する（ドキュメント、プレゼン、コード、デザイン等）

**実例:** frontend-design スキル

```
"高デザイン品質の本番グレードのフロントエンドインターフェースを作成する。
Webコンポーネント、ページ、アーティファクト、ポスター、アプリケーションを
構築するときに使用する。"
```

**主なテクニック:**
- スタイルガイド・ブランド基準を埋め込む
- 一貫した出力のためのテンプレート構造
- 完成前の品質チェックリスト
- 外部ツール不要 — Claude の組み込み機能を使用

---

### カテゴリ2: ワークフロー自動化

**目的:** 一貫した方法論が必要な複数ステップのプロセス（複数のMCPサーバー連携を含む場合も）

**実例:** skill-creator スキル

```
"新しいスキル作成のインタラクティブガイド。ユースケース定義、
frontmatter生成、手順作成、検証をステップごとに案内する。"
```

**主なテクニック:**
- バリデーションゲート付きのステップバイステップワークフロー
- 共通構造のテンプレート
- 内蔵レビュー・改善提案
- 反復改善ループ

---

### カテゴリ3: MCP拡張

**目的:** MCPサーバーが提供するツールアクセスにワークフローガイダンスを追加する

**実例:** sentry-code-review スキル（Sentry提供）

```
"Sentry の MCP サーバーを使って GitHub PR で検出されたバグを
自動解析・修正する。"
```

**主なテクニック:**
- 複数のMCPコールを順序立てて調整
- ドメイン知識を埋め込む
- ユーザーが毎回指定する必要のあるコンテキストを提供
- よくあるMCPエラーのハンドリング

---

## 5つのワークフローパターン

### パターン1: 順次ワークフロー

複数ステップを決まった順序で実行する必要があるとき。

```markdown
## ワークフロー: 新規顧客オンボーディング

### Step 1: アカウント作成
MCPツール呼び出し: `create_customer`
パラメータ: name, email, company

### Step 2: 支払い設定
MCPツール呼び出し: `setup_payment_method`
待機条件: 支払い方法の確認

### Step 3: サブスクリプション作成
MCPツール呼び出し: `create_subscription`
パラメータ: plan_id, customer_id（Step 1の結果）
```

**ポイント:** 明示的な順序・依存関係・各ステップでのバリデーション・失敗時のロールバック手順

---

### パターン2: 複数MCP連携

複数サービスにまたがるワークフロー。

```markdown
### Phase 1: デザインエクスポート（Figma MCP）
1. Figmaからデザインアセットをエクスポート
2. デザイン仕様書を生成

### Phase 2: アセット保存（Drive MCP）
1. Driveにプロジェクトフォルダを作成
2. アセットをアップロード

### Phase 3: タスク作成（Linear MCP）
1. 開発タスクを作成
2. アセットリンクをタスクに添付
```

**ポイント:** 明確なフェーズ区分・MCP間のデータ受け渡し・次フェーズ前のバリデーション

---

### パターン3: 反復改善

出力品質がイテレーションで向上するとき。

```markdown
## レポート生成

### 初期ドラフト
1. MCPでデータ取得
2. 初稿を生成

### 品質チェック
1. バリデーションスクリプト実行: `scripts/check_report.py`
2. 問題を特定:
   - 欠落セクション
   - フォーマット不一致

### 改善ループ
1. 各問題を修正
2. 該当セクションを再生成
3. 再バリデーション
4. 品質基準を満たすまで繰り返す
```

---

### パターン4: コンテキスト依存のツール選択

同じ結果を出すのに、状況によって使うツールが違うとき。

```markdown
## スマートファイル保存

### 判断ツリー
1. ファイルタイプとサイズを確認
2. 最適な保存先を決定:
   - 大きなファイル（>10MB）: クラウドストレージ MCP
   - 共同編集ドキュメント: Notion/Docs MCP
   - コードファイル: GitHub MCP

### 実行
決定に基づいて適切なMCPツールを呼び出す
```

---

### パターン5: ドメイン固有の知識

ツールアクセスを超えた専門知識をスキルに埋め込むとき。

```markdown
## コンプライアンスチェック付き決済処理

### 処理前（コンプライアンスチェック）
1. MCPで取引詳細を取得
2. コンプライアンスルールを適用:
   - 制裁リストを確認
   - 法域の許可を確認
   - リスクレベルを評価

### 監査証跡
- すべてのコンプライアンスチェックをログ
- 処理決定を記録
```

## スキルサイズガイドライン

| ファイル | 推奨サイズ |
|---------|-----------|
| SKILL.md | 500行以内、5000語以内 |
| references/ の各ファイル | 必要に応じて（制限なし） |
| scripts/ | 実行可能・ドキュメント付き |
| 同時有効スキル数 | 20〜50以内推奨 |
