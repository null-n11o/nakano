# 委任プロトコル — 共有スキル

KCP全体で共通して使う委任・コラボレーションのパターン集。

---

## Agent呼び出しの標準パターン

### 単一Agentに委任する場合
```
[Agent名].md の役割定義に従い、以下のタスクを実行してください:

タスク: [具体的な指示]
コンテキスト: [必要な背景情報]
参照ファイル: [読むべきファイルのパス]
期待する出力: [何を返してほしいか]
```

### 複数Agentを連携させる場合
```
以下の順番でタスクを処理してください:

Step 1: [Agent1] → [何をするか]
Step 2: [Agent2] → [Step1の結果を受けて何をするか]
Step 3: [Agent3] → [最終アウトプット]
```

## 横断的コラボレーションパターン

### CMO + CDO（マーケティング×デザイン）
コンテンツにビジュアルが必要な場合：
1. CMO → コンテンツ方針・メッセージ確定
2. CDO → ビジュアルデザイン実施
3. CMO → 最終確認・承認

### CTO + CDO（開発×デザイン）
プロダクト開発時：
1. CDO → UX/UIデザイン（Figmaで仕様化）
2. CTO → 技術実装
3. CDO → 実装確認（デザイン再現性チェック）

### CSO + CFO（営業×財務）
大型案件の受注判断：
1. CSO → 案件内容・要件整理
2. CFO → 収益性・リスク評価
3. CEO → 最終判断

---

## エスカレーションパス

```
Specialist → Chief（管轄外の判断・リソース追加が必要な場合）
Chief → CEO（予算重大影響・複数事業に影響・異常事態）
```
